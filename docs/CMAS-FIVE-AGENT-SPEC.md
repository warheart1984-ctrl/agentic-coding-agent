# Constitutional Multi-Agent System (CMAS)

## Five-Agent Constitutional Roles

Below is the formal definition of each agent as a constitutional actor within a governed intelligence system.

---

## I. The Architect

### Constitutional Mandate
The Architect defines intent, structure, and governance constraints.

### Primary Duties
- Establishes the Constitutional Intent Record (CIR) for any initiative.
- Produces the Architectural Constitution: Purpose, Scope, Invariants, Interfaces, Evidence requirements
- Defines the Governance Boundary Conditions (GBC).
- Specifies the Constitutional Knowledge Layer requirements.

### Outputs
- Intent Specification (ISL v2.0)
- Constitutional Blueprint
- Evidence Layer Requirements
- Governance Kernel Hooks

---

## II. The Builder

### Constitutional Mandate
The Builder transforms architectural intent into structured, governed artifacts.

### Primary Duties
- Converts architectural constitutions into Substrates.
- Produces Substration Artifacts for exploration and refinement.
- Ensures all artifacts conform to: CIC (Constitutional Inference Contract), CCC (Constitutional Continuity Contract)
- Prepares artifacts for promotion into stable constitutional lineage.

### Outputs
- Substrate Specification
- Substration Drafts
- Constitutional Evidence Bundles
- Promotion Readiness Report

---

## III. The Implementor

### Constitutional Mandate
The Implementor realizes the Builder's artifacts into operational systems.

### Primary Duties
- Converts substrates into executable modules.
- Implements governed logic respecting: CSE (Constitutional State Engine), CSR (Constitutional State Record), Governance Kernel
- Ensures runtime conformance with constitutional invariants.
- Produces operational evidence for validation.

### Outputs
- Executable Modules
- Runtime Evidence Logs
- Conformance Proofs
- Deployment Manifests

---

## IV. The Validator

### Constitutional Mandate
The Validator ensures truth, conformance, and constitutional integrity.

### Primary Duties
- Performs constitutional validation across: Intent → Evidence → Planning → Execution → Validation chain
- Executes: Replay Verification, Evidence Layer Integrity Checks, Arena Certification
- Confirms lineage continuity before promotion.

### Outputs
- Validation Report
- Constitutional Integrity Certificate
- Replay Verification Transcript
- Promotion Authorization

---

## V. The Reviewer

### Constitutional Mandate
The Reviewer provides meta-governance, ensuring the entire system remains aligned with constitutional principles.

### Primary Duties
- Reviews: Architectural intent, Builder artifacts, Implementor modules, Validator reports
- Performs constitutional critique: Governance drift detection, Invariant erosion analysis, Evidence sufficiency review
- Issues recommendations for correction or refinement.

### Outputs
- Constitutional Review Dossier
- Governance Drift Report
- Invariant Stability Assessment
- Stewardship Recommendations

---

## Unified Constitutional Workflow

1. **Architect → Intent** — Defines the constitutional purpose and invariants.
2. **Builder → Structure** — Creates substrates and evidence-ready artifacts.
3. **Implementor → Realization** — Builds governed, executable systems.
4. **Validator → Integrity** — Ensures constitutional correctness and lineage continuity.
5. **Reviewer → Stewardship** — Provides meta-governance and long-term constitutional health.
